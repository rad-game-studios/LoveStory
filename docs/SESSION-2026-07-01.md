## Per-segment length system (important)
Segments may set an optional `length` (defaults to `SEGMENT_LENGTH = 3600`).
`RunScene` computes cumulative starts in `computeLayout()` → `this.segStarts[]`
and `this.segLengths[]`. Use these helpers, **do not** reintroduce global
`START_X + i * SEGMENT_LENGTH` math:
- `segmentStartX(i)` → `segStarts[i]`
- `segmentIndexAt(x)` → which segment a world-x is in
- `spread(relX, i)` → maps authored 0..`AUTHORED_SPAN` to world-x within segment `i`
- `updateBackdrops()`, `checkpointX()`, `worldWidth`, `buildFloor` all use the arrays.

## Power-ups (`src/entities/PlayerMain.js` → `POWER_UPS`)
- `skateboard` speed+size · `beer` → **fireball shooter** (`shoot: 'fire'`) ·
  `shootingStar` → star shooter (speed bonus removed) · `ring` → +3 hearts ·
  `wings` → timed flight/invincible · `musicNote` → **laser shooter** (`shoot: 'laser'`).
- **All power-ups now grow you to the same size.** `MAX_SIZE_MULT` caps every
  size at the skateboard's `1.5`; `musicNote` was given `size: 1.5` so laser
  matches too.
- Shooting: player calls `scene.fireFireball` / `scene.fireStar` / `scene.fireLaser`
  based on `this.shootType`. Fireballs ricochet + destroy enemies (never hurt the
  player); lasers are fast straight beams.
- Item icon textures are generated at runtime in `RunScene.ensureItemTextures()`
  (`fire-tex`, `laser-tex`, etc.). The special-box → power-up mapping lives in
  `RunScene.grantSpecial()`; HUD hints in `POWER_HINT` / `POWER_ICON` at the top
  of `RunScene.js`.

## Gotchas (read these)
1. **Phaser 4.2.0 has no `preFX`/`postFX` `addGlow` API.** The earlier white-glow
   outline silently did nothing (`sprite.preFX` is `undefined`). That's why the
   indicator is now a red chevron. Don't reach for Phaser 3 FX.
2. **Per-segment length:** never assume all segments are `SEGMENT_LENGTH`. Use
   `segStarts`/`segLengths`/`segmentIndexAt`. The backdrop crossfade + checkpoints
   depend on this.
3. **Runtime textures:** many items/collectables have no art file — they're drawn
   to canvas at runtime (`RunScene.ensureItemTextures`, `Coin.ensureEmojiTexture`
   for emoji glyphs). Add new icons there.
4. **Emoji rendering** relies on the OS emoji font (Apple Color Emoji, etc.).
5. **Asset reload gotcha (still true):** after adding/replacing files under `art/`,
   restart the dev server and hard-reload (Cmd+Shift+R).
6. **Git:** a large batch of changes since commit `1bb3886` (power-ups, fireballs,
   point/ouch HUD, laser, party lengthening, chevron, scorecard, How-to screen)
   is **uncommitted**. Check `git status` and commit before major new work.
